A flexible, responsive search result item component built using the compound component pattern. This component is designed to display search results with various levels of detail and adapts to different container sizes.

## Features

- **Compound Component Pattern**: Flexible composition with individual sub-components
- **Responsive Design**: Automatically adapts to container size using CSS Container Queries
- **Accessible**: Proper ARIA attributes and semantic HTML
- **Customizable**: Each sub-component can be styled independently
- **TypeScript Support**: Full type safety with comprehensive interfaces

## Components

### Core Components

- `SearchResultItem` - Main container component
- `SearchResultItemIcon` - Icon container
- `SearchResultItemContent` - Main content area
- `SearchResultItemArrow` - Arrow indicator

### Header Components

- `SearchResultItemHeader` - Header information container
- `SearchResultItemTitle` - Document title
- `SearchResultItemDescription` - Document description

### Meta Components

- `SearchResultItemCategory` - Category information
- `SearchResultItemCategoryIcon` - Category with folder icon
- `SearchResultItemFooter` - Footer container
- `SearchResultItemMeta` - Meta information container
- `SearchResultItemMatchType` - Match type indicator (hidden on narrow containers)

### Text with Icon Components

- `SearchResultItemTextWithIcon` - Base component for text with icon
- `SearchResultItemAuthor` - Author with user icon
- `SearchResultItemDate` - Date with calendar icon
- `SearchResultItemReadingTime` - Reading time with clock icon

### Tag Components

- `SearchResultItemTags` - Tags container
- `SearchResultItemTag` - Individual tag
- `SearchResultItemMoreTags` - "More tags" indicator

## Props

All components accept standard HTML props and can be customized with className. The main `SearchResultItem` component also accepts:

- `asChild` - Whether to render as a child component (for composition)

## Responsive Breakpoints

The component uses CSS Container Queries with the following breakpoints:

- `@[300px]` - Very narrow containers
- `@[350px]` - Narrow containers (hides text with icons)
- `@[400px]` - Medium containers (hides match types)
- `@[600px]` - Wide containers (shows all content)

## Accessibility

- Proper semantic HTML structure
- ARIA data attributes for component identification
- Keyboard navigation support
- Screen reader friendly

## Styling

The component uses Tailwind CSS classes and follows the design system. All components can be customized by passing className props or using the `asChild` prop for complete customization.
